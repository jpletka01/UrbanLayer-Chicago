## Key Decisions
- **Conclusion-first (P0 Verdict Band).** Lead with the verdict, support with card-linked evidence, commit to ONE next step. **Deterministic, no LLM** — `lib/scorecardVerdict.ts` (`computeVerdict`: 6 categories — strong / incentive_driven / constrained / limited / entitlement_defined / insufficient_data) calibrated on 59 real parcels; `VerdictBand.tsx` renders it. Replaces the old facts-only flag line (which restated cards without concluding). Friction = the 7 parcel-specific overlay flags only; ARO + violations are citywide/area constants, NOT friction.
- **Scope discipline — the trust thread (#2/#3, then #4b).** parcel ≠ area; never present an area figure as the parcel's. Scorecard violations switched from `violations_by_community_area` (a trust bug — a whole neighborhood's count rendered as the parcel's) to address-exact (`_address_violations_data`), a **tri-state**: present / confirmed_zero ("no violations on record at this address") / unconfirmed (lookup couldn't run → omitted, NOT silent-same-as-zero). Crime/neighborhood demoted to a collapsed "Neighborhood context — describes {area}" section so area numbers can't read as parcel facts; page length now tracks verdict confidence (thin parcel → short page).
- **Grounding seam (#1).** (a) scope-tag every payload field (prompt rule 27 partitions THIS-PARCEL vs SURROUNDING-AREA, bans "this property has N violations"); (b) ship the distilled verdict + **caveats** into chat context (rule 28 forces hedging when caveated); (c) pin-null/nearest parcels ship **zoning-only** grounding and **OMIT** PIN-keyed property/comps (they may be a neighbor's — signed-off *omit*, not ship-and-caveat); (d) conversation-sticky grounding (`activeGroundingRef` in `useChat`) survives typed follow-ups (fixed the no-opts `sendMessage` leak); (e) staleness — distilled summary not the full response, and a **pivot guard** (`_PARCEL_HINT_PIVOT_MI=0.1`) + the pin-match grounding gate drop stale parcel context when a turn pivots to a different address.
- **#4b — address-violations into the grounding payload (the coherence finish of #2).** New `AddressViolations {status, summary}` (parcel-scoped, deliberately distinct from the area `violations` field; never merged). Frontend derives the same tri-state the page shows; ships in BOTH grounding tiers (address-keyed → identity-independent). Prompt **rule 29**: confirmed_zero ⇒ AFFIRM "no violations on record at this address" (don't retreat to the area count); unconfirmed ⇒ never say zero, never substitute the area figure. **confirmed_zero and unconfirmed are OPPOSITES** — a parse failure reading as zero is the violations bug reborn in the chat.
- **Subordination of secondary doors (#5/#6).** "Ask the analyst" (Home hero + PageHeader nav) and the Discovery→chat seam are LABELED but subordinate to the primary action; both open an **ungrounded** clean chat (`setGrounding(null)` on the `?ask=1` and `?q=`-no-pin paths — which also hardened the persona cards + failure-recovery that share that branch). **Discovery→chat is bare navigation (Track A)**, NOT a structured payload: a pre-filled area question answered by the chat's native area retrieval, so the area-as-parcel conflation can't enter a brand-new doorway. The free→paid wall became a **fork** (paid unlock OR free "ask the analyst about this area") — softens the cliff + cross-sells chat.
- **One money action (#4/#4c).** Collapsed 4 report entry points (ReportTeaser, VerdictBand secondary, sticky bar, ReportCTACard) to ONE (ReportCTACard); removed the redundant FinancialSnapshotStrip (every figure already lived in a card). azure = do work, terracotta = costs money.
- **#7 payoff preview — BUILT THEN REVERTED.** A "What you'll get" mini-verdict card on the empty-state Home, showing a real parcel's verdict + a labeled click-through. Removed after seeing it live: it showed a specific parcel's jargon (B3-2 / FAR 2.2 / landmark) to a context-less newcomer — the "you get a conclusion" signal was buried under terms only meaningful after a search; it crowded the hero; and the subline already makes the promise in plain words. **Lesson: green tests don't catch "heavier than it's worth" — only eyes on the live page do.**

## Process & deploy lessons (the reusable ones)
- **Verify what actually ships, in the form it ships.** (1) **Served-image over git-HEAD** — confirm the prod build rebuilt by polling the served asset *hash* + content, not the commit on main (a committed-but-not-rebuilt deploy is the failure mode where "it's on main" and "it's live" diverge). (2) **`npm run build` (= CI's `tsc -b`) over `tsc --noEmit`** — `tsc --noEmit` is weaker (misses noUnusedLocals); a leftover `useRef` import passed it but failed `tsc -b`, and since the deploy is CI-gated (`ci.yml`: `deploy: needs: test`), that build failure **silently skipped the deploy** while reporting nothing. Documented as the pre-push gate in both CLAUDE.md files.
- **Live-review as the gate, not the test suite.** Playwright against a running app + **measured WCAG contrast** caught the dark-mode azure-hover AA miss (4.20, sub-AA — the hover token was *lighter* than resting) that no eye and no unit test would. Tests confirm payload *shape*; only live model output confirms the chat actually affirms/scopes correctly under real Socrata.
- **Decouple deploys you can't jointly vouch for.** The theming overhaul (10 commits, not reviewed this session) was merged + **verified live on prod FIRST**, then the scorecard work rebased on top and shipped as a second, independently-revertible deploy. So when the 2nd deploy stalled, "which half broke" was immediately answerable.
- **A push to `main` is NEVER a no-op.** The deploy job runs `docker compose up -d --build`, which **recreates containers** → a brief origin-unreachable window → **Cloudflare 521 on every main push**. A docs-only push triggered one for zero app change. See known-issues "Operational Status" + the backlog below.

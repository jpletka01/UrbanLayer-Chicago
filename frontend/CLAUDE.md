# Frontend — UrbanLayer

## Stack

React + TypeScript + Vite + Tailwind v3. Map: Mapbox GL JS (dark-v11) + deck.gl via `@deck.gl/mapbox` MapboxOverlay. State: React hooks (no external state library). i18n: react-i18next with bundled JSON catalogs. Build: ~3.8MB main JS chunk / ~1.1MB gzip (mapbox-gl + deck.gl ride in it; html2canvas/jspdf are split), ~100KB CSS.

## Design Tokens

**Canonical design language: "Bento Pro"** (branch `feat/bento-pro`, 2026-06-30 →) — full spec in
`claude-context/guides/bento-pro-redesign.md` (token system + landing) and
`claude-context/guides/bento-pro-phase3-app-surfaces.md` (Scorecard/Discovery redesign, the
active working doc). The previous "Cyanotype on Vellum" system (azure accent, vellum neutrals,
Space Grotesk) is **retired**; `design-system.md` and `light-dark-theming.md` are historical.

**Theming (mechanics unchanged).** All color tokens are CSS-var-backed
(`rgb(var(--x) / <alpha-value>)`); class names are stable (`bg-dark-surface`, `text-text-primary`,
…) — only the backing flips. `:root`/`[data-theme="dark"]` + `[data-theme="light"]` hold the
palettes in `src/index.css`. `ThemeProvider`/`useTheme` own the preference (light/dark/**system**,
persisted as `urbanlayer-theme`); pre-paint script in `index.html` prevents FOUC. Mode-lock an
always-dark island with `data-theme="dark"` on its wrapper (hero, Footer) — NOT content sections
(they must flip).

**Palette = Bento Pro.** Near-black canvas (`#0a0a0a`) in dark / warm near-white in light;
separation via hairline borders (`border-dark-border` ≈10% white, `-strong` ≈20%), surfaces step
up in tiny increments. **Orange `#F9A474` is the single brand/action accent in BOTH modes**
(only small orange *link text* stays deeper `#c2410c` on light for AA). **Violet
(`text-highlight` #c3a3ff / `bg-highlight-fill` #6d4aef) = premium/costs-money ONLY.**
Dual-accent meaning: **orange = do the work, violet = costs money.** `state-*` tokens for
genuine state (bright -400 dark / -700 light); never raw emerald/rose/amber chrome.

**Action hierarchy**: primary = `bg-action` orange fill + `text-text-on-accent` (DARK label);
secondary = border + `text-link`; tertiary = `text-link`; premium = `bg-highlight-fill` +
`text-highlight-fg`. **Chip-buttons** (border-dark-border + caption + hover:accent) are the
page-action idiom — `InvestigateButton variant="chip"` (Scorecard), Discovery export/ask.

**Type**: same 10 named steps (`text-display / stat / section / subtitle / lead / title / body /
caption / micro / overline`). Fonts: `font-sans` Inter · `font-display` **Inter Tight** (scoped
to `.text-display`/`.text-section` via index.css) · `font-mono` **JetBrains Mono**.
Page-scale rule (Scorecard/Discovery): values `text-body`+, labels `text-caption` — `text-micro`
only for PINs/footnotes; the old sidebar-density micro-everywhere style is a chat-rail-only idiom.

**Radius**: cards/panels/modals `rounded-bento` (28px) / `rounded-bento-sm` (20px) — carried by
the Card/Modal primitives; controls/inputs `rounded-lg`; chips/badges `rounded-md`.

**Glow/shadow/ambient**: `shadow-glow` (accent halo on interactive cards), theme-aware
`shadow-card`/`shadow-modal`, `bg-brand-gradient` (orange CTA blend). `body::before` renders two
faint orange/violet corner blooms site-wide; `AccentRails` adds edge-masked orange plat-grid
rails to the landing's below-hero margins.

**Primitives** (`src/components/ui/`): `Card` (flex-col, `className` pass-through — equal-height
section grids stretch cards via `flex-1`), `Chip`, `Modal`. Use these; don't hand-roll chrome.

**Color discipline**: chrome = orange + neutral only; hue only for genuine state. Exempt
functional data encodings: `lib/mapColors.ts`, `DataPill`, `discovery/upsideColor.ts` (the
upside ramp also tints Discovery's list badges — same encoding as the map dots), map layers.

**Chart-color rule (2026-07-07)**: chart series 1 = **brand orange**, series 2 = a **neutral
gray step** — never blue/violet/rainbow (a validated-but-off-brand blue read as clip-art;
violet means costs-money). Identity may ride position + legend (a neutral's low chroma is
deliberate de-emphasis). State colors only for genuine good/bad (e.g. the assessed-value
▲/▼). Map encodings reuse mapColors/upsideColor unchanged; run the dataviz validator per
theme for any new multi-hue palette.

## Patterns

- **Per-message context**: each assistant message stores its own `context`, `plan`, `mapData`, `mapFetchedAt`. Citations survive multi-turn. Clicking a past user message loads that turn's data into the sidebar. **Per-message chips (2026-07-03)**: every completed assistant answer carries a `ContextChipStrip` (`Map · N / Data · N / Sources · N`, rendered via `MessageBubble`'s `footer` slot) that opens the panel/sheet on that tab **scoped to that turn** (reuses `handleMessageClick`; `"map"` falls back to the Data tab on desktop — `map` is a mobile-sheet-only SidebarView). Suppressed while streaming + on shared views. `countDataCategories` lives in `lib/contextSummary.ts`.
- **Sidebar (desktop)**: drag-to-resize, collapsed rail (44px). Data tab (map + analytics) / Sources tab (code chunks). Auto-opens when context arrives.
- **Sidebar (mobile)**: `MobileSidebarSheet` bottom sheet with 3 snap heights (20vh peek / 70vh default / 90vh full). 3-tab layout: Map / Data / Sources (bypasses `DataMapLayout`, renders `MapView` and `DataView` independently at full height). MapView stays mounted when switching tabs to preserve GL context. Smart default tab: Map for spatial queries, Data for domain queries, Sources for legal questions. `isMobile` prop on `MapView` drives mobile-specific rendering: ONE "Layers" button + popover holds ALL map controls (layer toggles + filters — no chip stack on the map face), tap-detail renders as a bottom-docked card (not a modal), 14px `pickingRadius`. The 20vh **peek snap shows a one-line summary strip** ("1143 crimes · …", tap → default) as an overlay — never replace sheet content or the GL context dies. The Data pane wraps `DataView` in `[data-density="comfortable"]` (micro→13px, sm→15px via the index.css scope — the type-axis sibling of the `[data-theme]` island idiom); desktop rail stays compact.
- **New sidebar card**: use `CollapsibleCard` pattern from `sidebar/CollapsibleCard.tsx`.
- **Chat→Scorecard bridge**: `sidebar/ScorecardBridgeCard.tsx` pinned at top of `DataView` when the active message's context resolves a parcel (`property.pin14` → `/scorecard?pin=`, else `resolved_address` → `?address=`). `ReportTeaser` takes optional `href` — clickable in the chat sidebar, static on ScorecardPage. Both fire `scorecard_bridge_click`. The word "report" is reserved for the paid Development Feasibility Report; the free chat export is the "transcript" (button: "Export"). **Scorecard→chat handoff (reverse direction, 2026-06-12)**: InvestigateButton takes optional `pin` → `/?q=...&pin=...` → App.tsx autosend passes `{parcelPin}` → `chatStream` sends `parcel_pin` → backend `_apply_parcel_hint` resolves the exact parcel instead of re-geocoding the question text. Read-only: chat never writes the selection (truth-model §3). **Scorecard chat grounding (2026-06-21, LIVE)**: the same handoff also ships `scorecard_context` (built by `lib/scorecardContext.ts` from the `ScorecardResponse` retained in `SelectedParcelContext`) so the backend reads pre-resolved facts and skips re-fetching them. Grounding rides the **explicit pin handoff only** — never free-typed composer turns (drift hazard at the router seam). The "Ask about this property →" entry → `/?pin=` (no `?q=`, no auto-send) sets `composing=true` to render the workspace, where `ChatInterface` shows flag-aware grounded starters that **send with the pin + address-embedded question** (a deictic "this lot" routes to clarification — the question must carry the address). Design + lessons: `claude-context/archive/2026-06-21_scorecard-chat-grounding.md`. **Deepened 2026-06-30 (LIVE):** grounding is now **conversation-sticky** — `useChat` holds `activeGroundingRef` (set once on the handoff, attached to EVERY turn) so typed follow-ups stay grounded (fixed the no-opts `sendMessage` leak); the distilled **verdict + caveats** ride into context; **pin-null/nearest** parcels ship zoning-only grounding and OMIT PIN-keyed property/comps (may be a neighbor's); and the **address-violations tri-state** (`address_violations`, built by `scorecardContext.ts` from the held response) ships so chat affirms "no violations on record at this address" instead of the area count. Backend drops stale grounding on a pivot (pin-match gate + `_PARCEL_HINT_PIVOT_MI`). `?ask=1` (Home/nav "Ask the analyst") + the Discovery seam open **ungrounded** (`setGrounding(null)`). See `claude-context/archive/2026-06-30_verdict-grounding-ux.md`. **Tier-2 extension (2026-07-03, LIVE):** `ScorecardContext.traffic` (nearest counted street, lat/lon-scoped → ships in BOTH tiers; CHRS/energy already ride `property.*`); the grounded empty-state starters are built by pure `propertyStarterKeys()` (lib/scorecardContext.ts, unit-tested) — priority-ordered `build, zoning, [chrs], [comparables], [traffic], [incentives], neighborhood` capped at 4, with the conditional slots firing on NOTABLE data only (CHRS orange/red = demolition-permit review; traffic ≥15k ADT). See `claude-context/archive/2026-07-03_chat-usability.md`.
- **Citations**: `[N]` → `CitationPill` (§ section reference), `[data:X]` → `DataPill` (opens Data tab).
- **Map layer order** (bottom → top): zoning polygons → overlay districts → incentive zones → parcel boundary → data dots (crime/311/permits) → transit stations → address pin.
- **Map layer defaults** (`MapView.tsx`, revised 2026-06-12 map-relevance review): zoning/incentives/overlays ON when present; transit OFF (ratchets on via `hasTransitContext`); crime/311/permit dots OFF. Dots auto-enable only two-factor: `deriveFilterMode(sources)` is single-source AND `pointsAreSubject(intent)` (incident_lookup/trend_analysis/event_query) — a legal question retrieving permits as context stays quiet. Backend gates the zoning quilt + overlay layers on `_landuse_map_relevant(plan)` (land-use vocabulary or site_due_diligence workflow), not `requires_disclaimer`. **The map doesn't render at all without spatial content**: `hasSpatialMapContent()` (mapColors.ts) gates MapView in `DataMapLayout` and the Map tab in `MobileSidebarSheet`; pure code-research turns get a full-height data panel. **Dense points cluster (2026-07-03)**: >300 filtered points below zoom 13 render as count-labeled cluster dots (hand-rolled grid clustering in `MapView.tsx` — Scatterplot+Text layers; ⚠️ deck.gl aggregation layers do NOT render in interleaved `MapboxOverlay` mode, verified empirically — don't retry HeatmapLayer). Zoom tracked via `zoomend`.
- **Tooltip**: `position: fixed` via `createPortal` to `document.body`, viewport-clamped with `useLayoutEffect`.
- **InfoTooltip**: Wrap term text in `<InfoTooltip term="key">{text}</InfoTooltip>` for hover/tap definitions. Uses `termDefinitions.ts` for lookups across overlays, zones, incentives, flood zones. Dotted underline trigger, 150ms hover persistence, click-away dismiss on mobile.
- **Charts**: `PieChart` (SVG donut) and `BarChart` (SVG horizontal bars) are custom — no chart library. `BarChart` takes `DistributionBucket[]` and renders labeled horizontal bars with hover state.
- **Routing**: `/` (splash — address-first hero via `HeroEntrance` → `/scorecard?address=`; librarian chat secondary), `/c/:id` (conversation), `/s/:shareToken` (shared read-only view), `/scorecard` (property scorecard, non-AI), `/explore` (**retired 2026-06-14 → redirects to `/discovery`**), `/discovery` (Property Discovery workbench — free teaser/top-10 + premium full list+map+export; **LIVE, nav-linked, full citywide — all 77 CAs**), `/pricing` (Free vs Pro plan comparison), `/admin` (dashboard, admin-only via `ProtectedRoute`), `/about` (technical deep dive), `/privacy` (privacy policy, English-only).
- **Auth**: `AuthProvider` wraps the app in `main.tsx`. **No gate on `sendMessage`** — anonymous chat works at the server-enforced 3/day IP limit. Anonymous = in-memory only: `canPersist` (`!authRequired || isAuthenticated`) gates conversation creation, history load, and attachments (the conversation endpoints 401 without a session); workspace header shows a "Sign in to save your research" nudge, and a 429 renders the server detail + sign-in button (`rateLimited` from `useChat`, `ChatStreamError` from `api.ts`). `AuthModal` appears at identity moments only (nudge, share, purchase, 429). `UserMenu` in workspace header shows avatar + sign-out. Admin link only visible when `user.tier === "admin"`. In dev mode (`GOOGLE_CLIENT_ID` not set), auth is fully bypassed — no sign-in UI shown.
- **401-interceptor**: `authFetch()` in `api.ts` intercepts 401 responses, attempts `POST /api/auth/refresh` via raw `fetch` (not `authFetch` to avoid recursion), coalesces concurrent refreshes with a module-level `_refreshPromise`, re-reads CSRF cookie after refresh, retries original request once.
- **History stripping**: `useChat.ts` sends only `{role, content}` in chat POST history (strips `context`/`plan`/`mapData` blobs) to avoid HTTP 413 from nginx's `client_max_body_size`.
- **Stream close detection**: `useChat.ts` tracks `receivedDone` flag — if the SSE stream ends without a `done` event and wasn't user-aborted, shows "Connection lost — please try again."
- **i18n**: `react-i18next` with 7 namespaces (`common`, `chat`, `sidebar`, `landing`, `map`, `data`, `pages`). Languages: English (default) + Spanish. Bundled JSON resources in `src/locales/{en,es}/`. Language persisted in localStorage as `urbanlayer-language`. `LanguageSelector` component in splash and workspace headers. Backend synthesizer responds in target language via `LANGUAGE_INSTRUCTION` prompt append — English path is +0ms latency. The **$25 Feasibility Report** is also localized (deterministic, no LLM): `fetchReport`/`createReportCheckoutSession` pass `language`; the backend renders via `backend/report_i18n.py` (see backend CLAUDE.md). Adding a new language: create `src/locales/{code}/` with 7 JSON files + add option to `LanguageSelector.tsx` + add entry to `LANGUAGE_NAMES` (`backend/synthesizer.py`) **and `MESSAGES` (`backend/report_i18n.py`)**. Non-React code (mapColors.ts, mapTooltip.ts, termDefinitions.ts) uses `i18n.t()` directly; scorecard/regulatory enum + zoning-qualifier strings go through `format.ts` helpers (`translateFloodSubtype`, `localizeZoningValue`). **Guards:** `src/i18n.parity.test.ts` asserts en/es key-set + interpolation-placeholder parity across all 7 namespaces (catches the drift class that caused the "3 of 5"/"3 of 3 zones" bug); `PageHeader.nav.test.ts` budgets nav-chrome strings ≤20 chars per locale (Spanish overflowed the workspace bar, 2026-07-03); backend `test_report_i18n.py` does the same for the report catalog. **Not yet localized**: Admin dashboard, About page (excluded by design).
- **Pipeline timing**: The `done` SSE event includes a `timings` dict (`PhaseTimings` in `types.ts`) with `conv_synth`, `router`, `retrieval`, `first_token`, `total` (all in ms). Logged to console via `[perf] pipeline timings (ms):` in `useChat.ts`.

## Docker / Nginx

- **Dockerfile**: multi-stage (node build → nginx serve). `NGINX_CONF` build arg selects config (defaults to `nginx.conf` for dev).
- **`nginx.conf`**: dev config — port 80 only, proxies `/api/`, `/chat`, `/health`, `/autocomplete`, `/section/` to `backend:8001`. SPA fallback for all other routes.
- **`nginx.prod.conf`**: production config — HTTP→HTTPS redirect on port 80, SSL termination on port 443 (Cloudflare Origin Certificate), security headers (HSTS, CSP tuned for Mapbox/deck.gl, X-Frame-Options DENY), gzip compression. Same proxy locations as dev plus `X-Forwarded-Proto`. `client_max_body_size 16m` on `/api/` and `/chat` location blocks.
- **CSP domains**: `script-src` allows `static.cloudflareinsights.com`; `style-src` allows `fonts.googleapis.com`; `font-src` allows `fonts.gstatic.com`; `img-src` allows `lh3.googleusercontent.com` (Google avatars); `connect-src` allows `*.ingest.de.sentry.io` (Sentry) + `*.clarity.ms`/`c.bing.com` (Clarity); `script-src` allows `www.clarity.ms`. CSP is defined in two places (lines 26 and 54) — both must be updated together.
- **Production deploy**: `docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d` — the override adds port 443, SSL cert volume mount, and selects `nginx.prod.conf`.
